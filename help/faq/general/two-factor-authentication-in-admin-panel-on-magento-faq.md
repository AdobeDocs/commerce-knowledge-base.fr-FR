# FAQ sur l’authentification à deux facteurs dans le panneau d’administration d’Adobe Commerce

1. **Qu’est-ce que l’authentification à deux facteurs dans le panneau d’administration ? Qu’est-ce qui a changé ?** L’authentification à deux facteurs (2FA) est une couche de sécurité supplémentaire pour vérifier votre identité, de sorte que seul vous puissiez accéder à votre compte d’administrateur, même si quelqu’un connaît votre mot de passe. Adobe a ajouté la prise en charge de 2FA dans le panneau d’administration de Commerce dans la version 2.3.0 afin de protéger le compte administrateur contre tout accès non autorisé. Dans la version 2.4.0, 2FA est activé par défaut sur le panneau d’administration et doit être configuré avant de se connecter à l’administration via l’interface utilisateur ou une API web. Adobe recommande vivement de ne pas désactiver le module 2FA.
1. **Pourquoi 2FA a-t-il été activé par défaut dans la version 2.4.0 ?** Fournir une couche supplémentaire d’authentification via 2FA est un paramètre sécurisé par défaut qui fournit une couche supplémentaire d’authentification pour rendre le portail d’administration plus sûr, ce qui réduit la surface d’attaque pour les attaques d’écrémage et le risque potentiel d’incidents de sécurité.
1. **Ce paramètre 2FA peut-il être désactivé dans la version 2.4.0 ?** Nous vous recommandons vivement de ne pas désactiver cette bonne pratique de sécurité et cette couche de protection supplémentaire avec une authentification à deux facteurs.
1. **Quelles versions prennent en charge 2FA ?La prise en charge d’** pour 2FA a été ajoutée dans la version 2.3.0, mais elle est activée par défaut dans la version 2.4.0.
1. **Comment activer 2FA sur 2.3 ?** Pour connaître les étapes d’activation de 2FA sur Adobe Commerce 2.3, reportez-vous à la section [Installer 2FA](https://developer.adobe.com/commerce/testing/functional-testing-framework/two-factor-authentication) dans notre documentation destinée aux développeurs.
1. **La 2FA pour l’administrateur sera-t-elle requise quels que soient l’adresse IP ou le réseau sur la carte ?** Dans la version 2.4.0, 2FA est activé par défaut et est requis indépendamment de l’adresse IP ou du réseau sur la carte. Il s’agit d’une bonne pratique en matière de sécurité que nous recommandons à tous nos clients de suivre. Vous trouverez des informations supplémentaires sur 2FA ici dans la section [Authentification à deux facteurs](https://developer.adobe.com/commerce/testing/functional-testing-framework/two-factor-authentication) de la documentation destinée aux développeurs.
1. **Je ne possède pas de smartphone, comment activer 2FA ?** 2FA sur le panneau d’administration prend en charge les authentificateurs suivants : authentificateur Google, Authy, clé U2F et Duo Security. Les utilisateurs administrateurs qui ne possèdent pas de smartphone peuvent utiliser une clé U2F ou des extensions de navigateur pour les authentificateurs tels que l’authentificateur Google.
1. **Pouvez-vous activer 2FA pour quelques utilisateurs seulement dans Adobe Commerce ou est-ce que tout le monde doit utiliser 2FA lorsqu’il se connecte ?** Adobe Commerce recommande à tous les utilisateurs administrateurs d’activer 2FA lors de la connexion à leur compte d’administrateur. Pour les webstores s’exécutant sur Adobe Commerce 2.4 et versions ultérieures, 2FA est activé par défaut pour chaque utilisateur.
1. **Quel est le comportement attendu lors de la configuration de 2FA sur le Panneau d’administration, puis-je utiliser le lien de l’e-mail reçu lors de la configuration et utiliser un navigateur différent de celui où je me suis déjà connecté et où j’ai demandé la configuration 2FA ?** L’utilisateur doit pouvoir ouvrir le lien e-mail dans n’importe quel navigateur, connecté ou non. Pour des raisons de sécurité, une seule session peut être ouverte au cours de ce processus et ils devront s’authentifier à nouveau s’ils changent de navigateur. L’avantage de 2FA est de forcer l’utilisateur à vérifier son identité à l’aide de deux méthodes différentes. Dans ce scénario de configuration, l’accès à l’e-mail de l’utilisateur n’est qu’une des méthodes, ce qui signifie qu’il doit toujours en fournir une autre. À ce stade du processus, il n’existe qu’une deuxième option, le mot de passe . Par conséquent, l’utilisateur ne peut pas utiliser le lien d’e-mail à lui seul pour se connecter et sera invité à saisir un mot de passe avant de pouvoir continuer s’il changera de navigateur ou s’il n’est pas déjà connecté.
1. **La configuration 2FA nécessite-t-elle une modification des paramètres ACL et accorde-t-elle l’accès aux utilisateurs non-administrateurs pour modifier les paramètres d’administration 2FA afin de permettre aux utilisateurs individuels de configurer leur 2FA personnel ?** Il existe deux ressources « Authentification à deux facteurs » [ACL](https://developer.adobe.com/commerce/php/tutorials/backend/create-access-control-list-rule/) dans l’interface. L’un autorise la configuration globale (**Magasins** > Paramètres > **Configuration** > **SÉCURITÉ** > **2FA**) et l’autre permet aux utilisateurs d’utiliser leur 2FA personnel (**Système** > **Permissions** > **2FA**). La configuration globale permet aux utilisateurs de type administrateur de configurer le système. Le second type de liste de contrôle d’accès est requis par les utilisateurs non-administrateurs pour se connecter et définir leur propre paramètre 2FA. La liste de contrôle d’accès qui permet une configuration globale permet à l’administrateur disposant de droits sur les paramètres de configuration de définir une configuration globale et n’a pas besoin que chaque utilisateur modifie les paramètres. Lorsque l’utilisateur s’est connecté et voit l’écran « Accès refusé », il peut consulter https://``<magento store>``/&lt;admin\_path>/tfa/tfa/requestconfig/ pour accéder à la configuration personnelle. Il s’agit d’un problème connu dans 2.4.1/2.3.6/2.4.0-p1 (package de sécurité 1.1.0) et qui sera résolu dans 2.4.2/2.3.7/2.4.1-p1 (package de sécurité 1.1.1).
